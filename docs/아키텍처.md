# 투자 필터 백엔드 — 아키텍처

**한 줄 정의:** DART·ECOS·KRX 3개 외부 API로 한국 상장사 ~2,850곳의 재무를 수집 → 지표 계산 → 2단계 필터·랭킹 → Django REST로 조회하는 **장기투자 종목선별 백엔드**.

> 이 문서는 상위(전체 조망)용이다. 디렉터리별 세부 규칙은 각 `CLAUDE.md` 참조.

## 2. 데이터 흐름 (수집 파이프라인)

```
종목코드.md
  → corp_code 변환(corp_code.py)
  → [orchestrator] 회사별:
       dart.py            연간 주요계정·지표 (fnlttSinglAcnt)
       dart_extractor.py  전체계정에서 CFO·CAPEX·현금·이자부채 (fnlttSinglAcntAll)
       dart.py            분기보고서
       ecos/krx           국채금리·시가총액
  → calculator.py         FCF·ROIC·WACC·EV·IC·Altman Z·Zmijewski·g 계산
  → filter.py             1차(영업이익/순이익/이익률/ROE) → 2차(ROIC≥WACC 등)
  → db.py                 Company + Yearly/Quarterly 저장
```

**조회 경로:** `URL → companies/views → service/db(+ranking/calculator) → models → JSON`

## 3. 핵심 설계 결정 (settings.py에 config화)

| 항목 | 값 | 위치 |
|---|---|---|
| 1차 필터 임계 | 영업이익률 ≥10%, ROE 규모별(8/10/12%) | `FIRST_FILTER` |
| 2차 필터 | ROIC−WACC ≥ 2%p | `SECOND_FILTER_ROIC_WACC_SPREAD` |
| 랭킹 가중치 | quality:price:growth = 1:1:1 | `RANKING_WEIGHTS` |
| 수집 병렬도 | 12 workers, API_DELAY 1.0s (≈12req/s) | `DATA_COLLECTION` |
| 계산 기본값 | 세율 25%, ERP 7%(+버퍼 0.5%p) | `CALCULATOR_DEFAULTS` |

## 4. 구조적 특징·주의점

- **`apps/service/`가 실질 코어** — 11개 모듈에 도메인 로직 집중. views·commands·orchestrator는 모두 이 레이어를 호출하는 얇은 껍질.
- **`apps/models.py`에 ORM 모델 + 순수 Python 데이터 클래스 혼재** (`YearlyFinancialDataObject` 등) — 수집 중 계산용 객체와 저장용 모델이 한 파일에 공존.
- **모델은 앱 루트(`apps/models.py`)에 단일 집중** — 앱별로 흩어져 있지 않음.
- **미계산=None 규약** — 지표 미산출은 0이 아니라 `None`(EV/IC 등)으로 구분 저장.
- **SQLite WAL 모드** — 병렬 수집 시 잠금 내성 위해 WAL·IMMEDIATE, save 재시도.

## 5. 데이터 소스

- **DART** (OpenDART): 재무제표·주요지표·분기보고서
- **ECOS** (한국은행): 국채 5년 금리 (WACC 무위험이자율)
- **KRX**: 시가총액 (당일 전종목 JSON 스냅샷 캐싱)
