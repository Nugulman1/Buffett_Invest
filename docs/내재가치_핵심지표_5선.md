# 내재가치 핵심지표 5선

> **축**: "이 사업이 **자본비용을 넘겨** 돈을 버는가 → 그 **가치 대비 지금 싼가**".
> 기업 내재가치 중심으로 역할별 **딱 하나씩**만 추린 표. 상세 정의·임계값은 [[지표_사전.md]] 참고.
> 작성 기준일: 2026-06-30.

## 역할별 1픽

| 역할 | 1픽 | 왜 이거 하나인가 |
|---|---|---|
| **수익성·해자(질)** | **ROIC−WACC 스프레드** | 회계이익이 아니라 자본비용을 초과해 번 수익(EVA). 내재가치의 본질 = "투입자본이 비용 넘게 버는가". 이 프로그램에서 가장 강한 무기이자 유일한 2차 필터(≥2%p). |
| **가격(저평가)** | **EV/IC** | 가격 축의 본질형. 투하자본 1원에 시장이 매기는 값 → ROIC−WACC(질)와 같은 분모(IC)를 공유해 "좋은 사업을 싸게"를 한 쌍으로 읽음. PBR보다 부채·현금 보정. **단, 현재 필터 미사용 — 트랙A 공백.** |
| **현금(이익의 진위)** | **FCF** | 회계이익은 부풀려도 현금은 덜 속음. 내재가치 = 미래 FCF의 현재가치라, 모든 절대가치 모델(S-RIM·EPV)의 원료. |
| **안정성·부실** | **Altman Z''-score** | 내재가치는 "망하지 않는다"가 전제. 다변량 파산예측 한 줄로 밸류트랩·적자필터 보완. 베타 불필요라 바로 도입 가능. |
| **성장** | **내재성장률 g (재투자율×ROIC)** | 후행 매출CAGR(현재 미적용) 대체. ROIC 기반이라 "재투자가 실제로 가치를 만드는 성장인지"를 봄 — 매출만 부푼 성장과 구분. |

**중심 한 쌍**: **ROIC−WACC(질) ↔ EV/IC(가격)** — 분모(IC)가 같아 "좋은 사업을 싸게"를 한 쌍으로 읽는다. 나머지 셋(FCF·Z''·g)은 그 판단이 무너지지 않게 받치는 보강재.

## 각 지표 한 줄 정의 (헷갈리기 쉬운 지점)

- **ROIC** = 투하자본 1원이 **버는 이익**(수익률, 높을수록 좋음). 분자는 회계 순이익이 아니라 **세후 영업이익**(EBIT×(1−세율)).
- **WACC** = 넘어야 할 **기준선**. (비싼 주주몫 Re × 비중) + (싼 빚 Rd × 비중 × 세금할인). **이자만이 아니라 "주주 요구수익까지" 섞은 평균** — 빚 안 쓴 회사도 주주 몫만큼은 벌어야 한다.
- **EV/IC** = 투하자본 1원에 **시장이 매기는 값**(배수, 낮을수록 쌈). ⚠️ ROIC(이익)와 혼동 금지 — 이건 **가격**이다. 분모 IC가 ROIC와 같아 쌍으로 본다.
- **FCF** = 영업현금(CFO)에서 **설비투자(CAPEX)까지 뺀** 잔액. 급여·세금·이자는 이미 CFO에서 빠짐. 이 남는 현금으로 배당·자사주·빚상환을 한다.
- **Z''-score** = 부도 경보기. 운전자본·이익잉여금·EBIT·자기자본을 총자산/부채 대비로 가중합. **2.6↑ 안전 / 1.1~2.6 회색 / 1.1↓ 부실.**
- **g** = ROIC × 재투자율(이익 중 사내유보 비율). **외부 자본 없이 스스로 크는 속도.**

## 안정성·부실 — Z'' + Zmijewski 둘 다 수집 (2026-06-30 확정)

경보등 용도(단독 탈락 없음)라 **둘 다 수집·표시하고 직접 써보며 고르기로 확정.** 입력 필드는 이미 모델에 다 존재 → 계산함수만 신규. 아래 비교표는 선택 근거(참고용). 부도예측은 4개 계열:

| 모델 | 연도 | 핵심 변수 | 방법 | 무형자산 기업 | 우리 환경 적합성 |
|---|---|---|---|---|---|
| **Altman Z''** | 1995 | 운전자본·이익잉여금·EBIT·자기자본 (총자산·부채 대비) — **4개** | 판별분석 | ✗ 헐거움(제조업 편향) | 베타 불필요, 즉시 도입 가능 |
| **Zmijewski X-score** | 1984 | ROA·부채비율·유동성 — **3개만** | 프로빗 | △ Z''보단 덜 편향 | ⭐ **가성비 최고** — DART로 즉시, 제조업 편향 없음 |
| **Ohlson O-score** | 1980 | 회사규모·부채비율·ROA·CFO/부채·순이익변화 등 — **9개** | 로지스틱 | △ 비제조 적합 | 변수 많아 수집부담, Zmijewski와 효익 겹침 |
| **Merton Distance-to-Default** | 1974 | **주가·주가변동성** + 부채 | 옵션이론 | ○ **이론상 최선**(시장가치 반영) | KRX 주가 보유라 계산 가능하나 변동성·옵션수학으로 복잡도↑ |

**판단 메모**:
- 회계기반 셋(Z''·Zmijewski·Ohlson)은 장부 자산을 분모로 써서 **무형자산 약점을 공유**. 정도만 다름(Z''가 제일 심함).
- 무형자산의 진짜 해법은 **Merton(시장기반)** — 단 연구도 "Merton조차 무형을 완전히는 못 잡는다" 경고. 복잡도 trade-off로 후순위.
- **현실적 1순위 후보 = Zmijewski** (Z''와 같은 회계기반인데 변수 3개·편향 적음). **무형 보완은 Merton**, KRX 주가 있으니 길은 열림.
- 어느 걸 골라도 **점수형 공통 한계**(예측력 1~2년 집중) → 단독 탈락이 아니라 **경보등**으로.

> 출처: deep-research / 웹검증(2026-06-30). Altman Z''는 1995 개정판(비제조·신흥국용)이라 1968 원본보다 이미 한 번 현대화된 식.

## ⚠️ 도입 전 주의 (대화에서 확인된 함정)

- **ERP는 7%로 정정 완료** (2026-06-30 코드 확인, `settings.py:173`, buffer 0.5%p 별도라 실효 7.5%). 과거 "10% 과대" 기록은 낡음 — [[지표_사전.md]] 4절·부록A·메모리 갱신 필요.
- **EV/IC·g는 랭킹으로 쓰기로 확정** (아래 「선별 적용 설계」). EV/IC는 계산됨(정렬만 연결), g는 신규 구현. 매출CAGR은 g로 대체 폐기.
- **Z''는 무형자산 집약 기업에 헐겁다** — 1950~60년대 제조업 기반이라 공장·재고 없는 테크·SaaS가 멀쩡해도 회색지대로 깎임. 우리가 노리는 성장주(무형 집약)와 충돌하므로 **단독 탈락이 아니라 "🔴 부실 구간 = 강한 감점/추가확인" 경보등으로만** 쓸 것.
  - 회계기반 대안 중엔 **Zmijewski(3변수, 제조업 편향 없음)**가 가성비 우위. 무형자산의 진짜 해법은 **주가 기반 Merton Distance-to-Default**(KRX 주가 보유라 계산 가능하나 복잡도↑, 후순위).
- **점수형 지표(Z 등) 공통 한계**: 예측력이 1~2년에 집중 → 리밸런싱 주기 설계가 성과를 좌우.

## 선별 적용 설계 — 결정 확정 (2026-06-30)

> 5선을 "어떻게 쓸지" 확정한 섹션. **코드 반영은 별도 세션**, 여기선 결정·아젠다만. 결정 주체: 사용자(LLM 권고 반영).

### 출력 구조: 필터 → 랭킹 → 경보 3층

기존엔 **필터(통과/탈락 이분법)만** 있었음. 여기에 **랭킹(통과 종목을 점수로 줄세움)을 신설**하고 경보(플래그)를 얹는다. 저평가·성장은 "선 넘었나"가 아니라 "얼마나 싸고 잘 크나"의 정도 문제라 자르지 않고 줄세우는 게 맞기 때문.

| 지표 | 역할 | 설정 |
|---|---|---|
| **ROIC−WACC** | **하드필터 + 랭킹** | 필터 최소선 ≥2%p 유지 / 랭킹은 초과분으로 점수 (중심쌍을 랭킹에서 재결합) |
| **EV/IC** | **랭킹** | 쌀수록 상위 (가격 축) |
| **g** (재투자율×ROIC) | **랭킹** | 잘 클수록 상위. 성장 축을 g로 일원화(매출CAGR 폐기) |
| **FCF** | **경보(플래그)** | 다년 음수 = 이익 진위 의심 표시 |
| **Z'' · Zmijewski** | **경보(플래그)** | 둘 다 수집·표시. 임계: Z'' 2.6/1.1, Zmijewski 부실확률 0.5 |

### 랭킹 설정
- **점수화 = 순위합산(마법공식식)**: 각 지표 등수를 더해 합이 최소인 종목이 1위. 등수만 쓰니 단위·극단값(소형주 EV/IC)에 강건, 한국 검증.
- **가중 = 질·가격·성장 동등(1:1:1)으로 시작**, 가중치는 config로 빼서 실험 조정. 정체성(가치 중심 vs GARP)은 못 박지 않고 돌려보며 찾음.
- **경보 개입 = 플래그만(순위 비개입)**: 부실/현금이상은 🔴 표시만 하고 순위엔 안 섞음. 두 부실모델(Z''·Zmijewski)을 나란히 비교·관찰하기 위함. 감점은 나중에 도입 가능.

### 1차 필터
- **임계값 config화 확정**: 현재 `filter.py` 하드코딩(ROE 8/10/12%·영업이익률 10%·적자 ≤1회) → `settings.py`로 이관해 실험 가능하게. 초기값은 현행 유지. "GARP 정체성과 충돌(무형 성장주 조기탈락)" 여부는 실험으로 검증.
- 성장 축은 **g(랭킹)로 일원화** → 1차 필터엔 성장 하드컷 없음(매출CAGR 폐기).

### 데드코드 정리 — 분류 (코드 추적 기반)

**남김(활성)**: ROIC·WACC·ROIC−WACC / EV·IC·EV/IC / FCF / 1차필터 입력(영업이익·순이익·영업이익률·ROE·총자산) / 핵심입력(이자부채·현금·CFO·CAPEX·이자비용·비지배지분) / 배당성향·연속배당연수·배당금·판관비율 *(선별 미사용이나 프론트 표시 유지 — 사용자 결정)*

**신규 구현(입력 이미 보유, 추가 수집 불필요)**: g(재투자율×ROIC) · Altman Z''(운전자본·이익잉여금·EBIT·시총·부채) · Zmijewski(ROA·부채비율·유동비율)

**밀 것(완전 미사용)**:
- `profit_before_tax` 필드 — DART 수집만, 미사용
- `capital_stock` 필드 — DART 수집만, 미사용
- `calculate_cagr` + 매출CAGR 필터(`filter_revenue_cagr`) — g로 대체 폐기

**정정**: ICR(`calculate_icr`)은 코드에 함수 자체가 없음(검색 0건) — 지표사전 함정 기록이 낡음. 밀 대상 아님.

### 새 세션 코드 아젠다
1. **1차 필터 5개 임계값 config화** (`filter.py` → `settings.py`)
2. **랭킹 레이어 신규 구축**: 순위합산·동등가중·가중치 config
3. **g 계산 신규** + 랭킹 연결 / **EV/IC 랭킹 연결**(이미 계산, 정렬만)
4. **Z''·Zmijewski 계산 신규** + 플래그 / **FCF 다년음수 플래그**
5. **데드코드 제거**: `profit_before_tax`·`capital_stock`·`calculate_cagr`·`filter_revenue_cagr` (삭제 전 의존성 재확인)
6. **낡은 문서 갱신**: ERP 10%→7%(지표사전 4절·부록A), ICR 함정 기록 삭제
