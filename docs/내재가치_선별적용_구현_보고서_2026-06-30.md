## LLM이 사용자 대신 내린 설계·결정 (대안 포함)

- **[DEC-1] Z'' X4 = 책값 자기자본**(시총 아님). 정본 §94 "시총" 표기는 §16·§29·§40 "자기자본"·임계값(2.6/1.1)·Altman Z''(1995) 정식과 모순 → 다수+정식 채택. *대안*: 시총(원본 Z-score) — 거부(Z'' 정식·임계 불일치).
- **[DEC-2] Zmijewski 부채축 = 총부채/총자산**(프로젝트 `debt_ratio`=총부채/자본 아님). 정식 계수 −4.336/−4.513/+5.679/+0.004, 부실확률=로지스틱, 플래그 P≥0.5(프로빗/로짓 0.5 동일). *대안*: 기존 debt_ratio 재사용 — 거부(정식 변수 정의 위반).
- **[DEC-3] g = ROIC×유보율**, 유보율=1−(배당금/순이익), [0,1] 클램프, 순이익≤0→None, 배당금None→유보율1. *대안*: ROE×유보율(지속가능성장 표준) — 거부(정본이 ROIC 명시).
- **[DEC-4] FCF 플래그 = 최근 3년 중 음수 ≥2회**(lookback/min config 인자). 정본 "다년 음수"의 구체화.
- **[DEC-5] 랭킹**: 경쟁순위(동점 동순위·등수 건너뜀), None축=최하위 등수 N, 부분키 weights는 누락축 1.0 폴백, score 최소=1위. 정본 "순위합산"의 동점·결측 처리 구체화.
- **[DEC-6] filter_revenue_cagr 전체 폐기**(필드+API+배지+마이그레이션). *대안*: 필터 로직만 제거하고 필드 유지 — 거부(필드가 배지에 묶여 전원 '미통과' 오작동). 안전근거: `dart.py:172` hasattr 가드가 객체필드 제거 시 setattr 스킵.
- **[DEC-7] 랭킹/플래그 라이브 통합은 후속 분리**(위 '범위밖' 참조). cross-sectional·벌크로드·페이지네이션 충돌로 단위테스트 부적합.
- **[DEC-8] L1 하드닝 자가발의**: 레드팀이 부분 FIRST_FILTER → KeyError 지적. 이 config의 목적이 '실험용 부분 조정'이라 기본값 병합 폴백으로 수정(테스트 3건 추가 박제). *대안*: 무시(런타임 데이터로 도달 불가) — 거부(실험 의도와 정면 충돌).
- **[DEC-9 / L2] 랭킹 weights 음수/0 미검증은 유지**. 0은 정당한 실험 노브(축 끄기), 음수는 operator 오류 → 코드검증 추가 안 함, 위험지점으로만 surface.
