# 트랙B 리팩터 코드리뷰 (LLM 복붙 → DART 자동계산)

날짜: 2026-06-26 · 대상: `git diff HEAD` (uncommitted, origin/main 기준) · effort: xhigh

## 핵심 문제 한 줄

이번 변경의 목적은 **2차 필터(ROIC−WACC) 자동화**인데, **WACC 입력(국채수익률·이자부채)이 조용히 0/과대로 들어가 스크리닝 결과를 뒤집는** 경로가 새로 생겼다. 투자 판단을 바꾸는 무증상 오류라 가장 위험하다. 리팩터 구조 자체(뷰→service 분리 T10, 락 일원화 T9, equity 프로퍼티 T8)는 안전 확인. 문제는 **재무 정확성** 쪽에 몰림.

## 🔴 Critical — 스크리닝 결과를 직접 왜곡

### 1. 국채수익률 실패 시 WACC가 risk-free=0%로 계산. `BOND_YIELD_FALLBACK_PCT`(3.5)는 정의만 되고 미사용
- 위치: `config/settings.py:181`, `orchestrator.py:99·150`, `calculator.py:240`
- 메커니즘:
  - 폴백 3.5%를 새로 넣고 주석은 "매직넘버 3.5 일원화(T6)"라 했으나 **전 코드에서 참조 0건**(grep 확인).
  - `orchestrator.py:150` ECOS 빈 응답이면 `collect_bond_yield_5y()`가 예외 없이 `0.0` 반환 → `yield_value=0.0` 저장 + `collected_at=now` → **0%가 24h "신선한 값"으로 캐싱**.
  - `orchestrator.py:99` `bond_yield_pct=(bond_yield_decimal or 0.0)*100=0` → `cost_of_equity=(0+0.5+10)/100=0.105` (정상: `(3.5+0.5+10)/100=0.14`).
  - WACC가 `자기자본비중×3.5%p ≈ 2.1~3.2%p` 과소 → **2차 필터 임계 0.02(2%p)보다 큼** → 떨어졌어야 할 기업이 `passed_second_filter=True`.
  - 트리거 흔함: 월초엔 ECOS가 전월 5년물 미게시 → 빈 응답 잦음.
- 고치는 법: `_fill_advanced_indicators`에서 `bond_yield_decimal`이 0/falsy면 `settings.BOND_YIELD_FALLBACK_PCT` 사용하도록 배선. (의도한 일원화 미완성)

### 2. ECOS 예외 시 이미 로드한 유효 캐시값을 버리고 0.0 반환
- 위치: `orchestrator.py:154`
- `_ensure_bond_yield` 바깥 `except`가 `return 0.0`. 직전에 `bond_yield_obj.yield_value`(과거 정상값)를 DB에서 읽어 스코프에 갖고 있는데도 버림. 네트워크 한 번 끊기면 배치 전체가 risk-free=0% → #1과 동일한 필터 오통과.
- 고치는 법: `except`에서 `return bond_yield_obj.yield_value or 0.0` 또는 폴백 상수.

### 3. dart_extractor 이자부채가 부분문자열 키워드 매칭으로 과대집계
- 위치: `dart_extractor.py:78-90, 152-160`
- `_is_debt_row`가 `any(k in nm)` (k=차입금/사채/리스부채)로 BS 행을 잡고 `+= abs(v)` 누적. 두 가지 오집계:
  - **차감계정(contra) 가산:** `사채할인발행차금`(사채 차감, 음수)이 "사채" 포함 → `abs(-20억)`이 더해져 사채를 순액보다 +40억 부풀림.
  - **합계+내역 중복:** `차입금및사채`(소계) + `단기차입금` + `사채`를 한 응답에 다 표기하면 소계가 한 번 더 더해져 2배.
- 이자부채는 WACC 부채가중치·ROIC/IC 분모·EV에 모두 들어가 파급 큼.
- 고치는 법: leaf 계정만 합산(소계/contra 제외), 또는 account_id 우선·키워드는 화이트리스트로 축소.

### 4. 이자비용·현금·NCI가 "못 찾으면 0"이라 해당 연도가 제외 안 됨 (이자부채만 키워드 폴백 — 비대칭)
- 위치: `dart_extractor.py:140-149`
- 이자부채는 키워드 폴백으로 잡히지만 이자비용은 표준 account_id 3개 중 없으면 `0` 유지(`None` 아님). `cost_of_debt=0/이자부채=0` → WACC 과소. 값이 `0`(≠`None`)이라 `orchestrator.py:103`의 연도 제외 조건을 통과해 **틀린 WACC가 저장·필터 사용**. 비표준 계정 태깅 기업에서 발생. (현금 누락→EV 과대, NCI 누락→EV 과소도 같은 0-vs-None 구조)
