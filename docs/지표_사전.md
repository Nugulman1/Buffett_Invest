## 1. 구현된 지표 [코드가 실제 계산·저장]

> 공식은 모두 `apps/service/calculator.py` 현물 기준. 임계값은 `apps/service/filter.py`.
> 비율 지표는 **소수로 저장**(0.105 = 10.5%), 프론트에서 ×100 표시.

### 1-1. 수익성 / 효율

#### ROE (자기자본이익률)
- **공식**: 당기순이익 / 자기자본. (1차 소스는 DART 주요재무지표 코드 M211550, 없으면 이 공식으로 폴백)
- **의미**: 자기자본을 얼마나 효율적으로 굴려 이익을 냈나. 높을수록 좋음. 한국 상장사 10년 평균 ~8%.
- **임계값 (규모별 평균 ROE)**: 대형(총자산≥10조) ≥ **8%** / 중견(5천억~10조) ≥ **10%** / 중소(<5천억) ≥ **12%**. 소형주를 더 높게 요구.

#### 영업이익률 (operating_margin)
- **공식**: 영업이익 / 매출액.
- **의미**: 본업의 마진. 높을수록 가격결정력·비용효율 양호. 기본 API에 없어 프로그램이 계산.
- **임계값**: 최근 5년 **평균 ≥ 10%**.

#### ROIC (투하자본수익률)
- **공식**: (영업이익 × (1 − 법인세율)) / (자기자본 + 이자부채 − 현금및현금성자산).
- **의미**: 사업에 실제로 투입된 자본이 세후로 얼마를 버나. **경제적 해자 판별의 핵심**. 회계이익이 아니라 투하자본 대비 수익이라 자본구조 왜곡이 적음.
- **임계값**: 단독 임계는 없고, 아래 ROIC−WACC 스프레드(2차 필터)로 쓰임.

#### WACC (가중평균자본비용)
- **공식**: (E/(E+D)) × Re + (D/(E+D)) × Rd × (1 − 법인세율)
  - E = 자기자본, D = 이자부채
  - **Re(자기자본비용)** = 국채5년 수익률 + **0.5%p(보수적 가산)** + 주주기대수익률(ERP)
  - **Rd(타인자본비용)** = 이자비용 / 이자부채
- **의미**: 이 회사가 자본(주주+채권자)을 끌어다 쓰는 평균 비용. 사업이 이걸 넘겨야 가치 창출.
- **임계값**: 단독 임계 없음. ROIC와의 차이로 판단(아래).

#### ROIC−WACC 스프레드 (2차 필터)
- **공식**: 최근 3년 평균 ROIC − 최근 3년 평균 WACC. (roic/wacc 없는 연도는 제외)
- **의미**: **자본비용을 초과해서 번 수익 = 경제적 부가가치(EVA)의 핵심.** 이 프로그램에서 가장 강한 무기. >0이면 가치를 창출하는 회사.
- **임계값**: ≥ **0.02 (2%p)** 이면 통과 (`SECOND_FILTER_ROIC_WACC_SPREAD`, 기본 0.02).

### 1-2. 현금 / 가치(가격 축)

#### FCF (자유현금흐름)
- **공식**: CFO(영업활동현금흐름) − |유형자산취득 + 무형자산취득|. (CAPEX는 절대값 — 처분 음수 방어)
- **의미**: 영업으로 번 현금에서 설비투자 빼고 진짜 남는 현금. **이익의 질**을 보는 재료(회계이익은 부풀려도 현금은 덜 속음).
- **임계값**: 필터 미사용 (계산만 함). → [함정](#4-️-함정-알려진-불일치버그) 참고.

#### EV (기업가치)
- **공식**: 시가총액 + 이자부채 − 현금및현금성자산(**전액**) + 비지배지분.
- **의미**: 회사를 통째로 인수할 때 드는 값. 시총만이 아니라 빚을 떠안고 보유현금은 빼는 **가격 축**.
- **임계값**: 단독 필터 없음. (과거 현금 70%만 차감하던 로직은 현재 전액 차감으로 통일됨 — [함정](#4-️-함정-알려진-불일치버그) 참고.)

#### IC (투하자본)
- **공식**: 자기자본 + 이자부채 − 현금및현금성자산. (ROIC 분모와 **동일**)
- **의미**: 사업에 실제로 묶인 자본. 단위는 원.
- **임계값**: 필터 미사용.

#### EV/IC
- **공식**: EV / IC.
- **의미**: 투하자본 1원당 시장이 매기는 가격. 낮을수록 싸게 거래되는 셈(가격 축). PBR의 EV판 사촌.
- **임계값**: 계산해서 보여주기만 하고 **필터엔 미사용** (트랙 A에서 채울 공백).

### 1-3. 안정성 / 배당

#### 부채비율 (debt_ratio)
- **공식**: 부채총계 / 자본총계 (**표준 정의**). 값이 **클수록 부채 의존도 높음(위험)**.
- **의미**: 자본 대비 빚이 얼마나 되나. 자본잠식(자본총계 ≤ 0)이면 계산 안 함(None). 무차입(부채 0)은 0.0으로 유효.
- **임계값**: 필터 미사용.

#### 배당성향 (dividend_payout_ratio)
- **공식**: 배당금지급 / FCF. (**표준은 배당/순이익** — 이건 변형. [함정](#4-️-함정-알려진-불일치버그) 참고.)
- **의미**: 벌어들인 자유현금 중 배당으로 나간 비율. FCF ≤ 0 또는 배당 정보 없으면 None.
- **임계값**: 필터 미사용.

#### 연속배당연수
- **공식**: 최신 연도부터 `배당금지급 > 0`인 해를 연속으로 카운트, 끊기면 멈춤.
- **의미**: 배당의 꾸준함. 길수록 안정적 주주환원.
- **임계값**: 필터 미사용.

#### 판관비율
- **공식**: DART 주요재무지표에서 '판관비율'/'판매비와관리비율' 수집 (직접 계산 아님).
- **의미**: 매출 대비 판매·관리비 비중.
- **임계값**: 필터 미사용.

### 1-4. 성장 / 적자 거르기

#### 매출액 CAGR
- **공식**: (최종연도 매출 / 시작연도 매출)^(1/years) − 1. (years = 데이터 수 − 1)
- **의미**: 복리 기준 연평균 매출 성장률. 중간 변동을 평활화한 성장 추세.
- **임계값**: ≥ **10%** — 단 **현재 미적용**(코드에서 통과로 고정). → 지금 필터엔 성장 축이 없음. [함정](#4-️-함정-알려진-불일치버그) 참고.

#### 영업이익 / 당기순이익 (적자 필터)
- **공식**: 영업이익 ≤ 0인 연도 수 / 당기순이익 5년 합계.
- **의미**: 적자기업 거르기. **이익의 부호만** 봄(질·현금화 여부는 미검증).
- **임계값**: 영업이익 ≤ 0 연도 **≤ 1회**, 당기순이익 최근 5년 **합계 > 0**.

### 1-5. 미사용 구현 지표

#### ICR (이자보상비율)
- **공식**: 영업이익 / 금융비용. (배수)
- **의미**: 영업이익으로 이자를 몇 배 감당하나 — 부실위험 판별. 높을수록 안전.
- **상태**: **현재 미사용.** 게다가 코드가 모델에 없는 필드를 참조해 호출 시 에러 → [함정](#4-️-함정-알려진-불일치버그) 참고.

## 2. 밸류에이션·해석 지표 [미구현 · 참고]

> 아직 코드가 계산하지 않음. `docs/저평가주_탐색_리서치.md`의 해석 가이드 요약. "어떻게 읽는가"를 공부할 때 보는 섹션.

#### PBR (주가순자산비율)
- **공식**: 시가총액 / 자기자본 (= 주가 / BPS).
- **의미**: 한국의 **사실상 표준 밸류 지표.** "진짜 싼 것"을 가장 직접 포착. 한국 평균 ~1.2배(선진국의 52% 수준).
- **해석**: 단독 신뢰 금지. **수준값 + 전년대비 증분**을 함께 봐야 밸류업 평가와 정렬.

#### PER (주가수익비율)
- **공식**: 시가총액 / 당기순이익 (= 주가 / EPS).
- **의미**: 이익 1원에 시장이 매기는 배수.
- **해석**: 싼 주식 발굴엔 PBR보다 덜 직접적. **사이클 고점 함정** 주의 — 경기민감 업종은 업황 고점에 PER이 가장 낮아 "가장 비쌀 때 가장 싸 보임".

#### EV/EBITDA · EV/EBIT (이익수익률)
- **공식**: EV / EBITDA(또는 EBIT). EV/EBIT의 역수가 이익수익률.
- **의미**: 감가상각 큰 자본집약 업종(철강·화학)에서 PER보다 신뢰성 ↑ (감가상각 더하고 순부채 반영해 자본구조 보정).
- **해석**: **낮을수록 저평가**.

#### PSR (주가매출비율)
- **공식**: 시가총액 / 매출액.
- **의미**: 적자기업도 평가 가능(매출은 양수).
- **해석**: **낮은 PSR ≠ 저평가.** 부채 많거나 수익성 낮으면 위험이 반영돼 낮게 나옴.

#### S-RIM (잔여이익모델)
- **공식**: 적정가치 = 자기자본 + 초과이익 / 요구수익률, 초과이익 = 자기자본 × (ROE − 요구수익률).
- **의미**: 미래 초과이익을 할인한 **절대가치**. 배수가 아니라 "적정가치 대비 현재 주가"를 한 줄로 줌.
- **해석**: 입력(BPS·ROE·자기자본)은 DART로 충당, 요구수익률만 ECOS 금리. 주가 < 적정가치면 저평가.

#### 마법공식
- **공식**: 이익수익률(EBIT/EV) + 자본수익률(ROC) 각각 독립 랭킹 → 순위 합산 최소값 선정.
- **의미**: 싸면서(이익수익률) 좋은(자본수익률) 회사를 동시에 잡는 조합.
- **해석**: 한국에선 **소형주(시총 1~3분위)에서만 통계적 알파** 유효, 대형주는 희석. ROC보다 **GP/A가 한국 데이터에서 더 우수**.

#### 밸류트랩 회피
- **공식**: 경기민감 업종은 1년 이익이 사이클 함수 → **5~10년 평균으로 정상화**해서 배수 재계산.
- **의미**: 싸 보이지만 구조적으로 쇠퇴하는 함정 회피.
- **해석**: **다년 적자 + 설비 축소 + 저PBR 동시 = 밸류트랩 플래그(강한 감점/제외).** 단일 연도 저PER·저PSR은 신호로 신뢰 금지.

| 지표 | 공식 | 의미 / 왜 추천 |
|---|---|---|
| **GP/A** (1순위) | 매출총이익 / 총자산 | 손익 맨 위라 조작 여지 적고 ROE·영업이익률과 직교. 한국 백테스트 단일 퀄리티 1위. 추가 수집 0. |
| 발생액 (Accruals) | (순이익 − CFO) / 평균총자산 | "현금 안 들어오는 이익" 적발 = 이익의 질. **저발생액일수록 양호**(Sloan). |
| 순부채/EBITDA | (차입금 − 현금) / (영업이익 + 감가상각) | 상환능력 정규화, 솔벤시 글로벌 표준. 현 부채비율의 업종편차 보완. |
| 현금전환율 | CFO / 영업이익 | 영업이익이 현금으로 뒷받침되나 — 흑자도산 조기경보. |
| 총주주수익률 | (배당 + 자사주순매입) / 시총 | 코리아 디스카운트의 37%가 주주환원. ROE>요구수익률인데 환원 낮으면 "일시적 저평가(매수후보)". |
| 내재성장률 g | 재투자율 × ROIC | 미적용 매출CAGR(후행) 대체. 성장의 질. |
| 자산성장률 | 총자산 증가율 | 과잉투자·M&A로 자산만 불린 기업 회피(**저자산성장**이 양호). |
| Piotroski F-score | 9개 항목 합성점수 | 개별 퀄리티를 묶어 노이즈 감소. 고F 그룹 초과수익. |

## 4. ⚠️ 함정 (알려진 불일치·버그)

> 지표를 잘못 읽거나, 코드와 표준이 다른 지점 모음. **stale 관리 지점** — 코드 점검 시 여기부터 본다.
> (✅ = 코드 현물 확인상 이미 해결, ⚠️ = 현재도 유효)

- **✅ 부채비율 역수 — 해결됨**: `calculate_debt_ratio`는 현재 표준 `부채총계/자본총계`로 계산한다(`calculator.py`). *잔여*: `calculate_basic_financial_ratios`의 **도크스트링**에 옛 설명("자본총계/부채총계")이 남아 있어 코드와 불일치 — 주석만 낡음.

- **✅ EV vs IC 현금 처리 불일치 — 해결됨**: 과거 EV는 현금 70%만, IC/ROIC는 전액 차감해 EV/IC 분자·분모 정의가 어긋났다. 현재 EV도 **현금 전액 차감**으로 통일(`calculate_ev` 도크스트링 "ROIC/IC 분모와 동일하게 통일 — T5"). *남은 정교화 과제*: 운용현금/초과현금 구분(초과현금만 차감)은 후순위.

- **⚠️ ICR — 호출 시 에러**: `calculate_icr`가 `yearly_data.finance_costs`를 참조하는데 모델엔 그 필드가 **없다**(`interest_expense`만 존재). 지금은 **미사용**이라 안 터지지만, 호출하는 순간 AttributeError. 살리려면 `interest_expense` 기준으로 고치고, 1년치는 일회성에 흔들리니 **다년 평균 ICR(정상화)** 권장.

- **⚠️ ERP(주주기대수익률) 불일치 & 과대**: 실제 설정값은 **10%**(`settings.EQUITY_RISK_PREMIUM`)인데 calculator 도크스트링엔 "5.0 등"으로 적혀 코드와 불일치. 또 10%는 한국 통상치(6~8%)보다 높아 WACC를 부풀려 ROIC−WACC 스프레드를 보수적으로(통과 어렵게) 만든다.

- **⚠️ 배당성향 = 배당/FCF (비표준)**: 표준 배당성향은 배당/순이익. 이 코드는 배당/FCF라 **FCF가 음수면 의미가 깨진다**(음수 비율). 해석 시 표준과 다른 값임을 기억.

- **⚠️ 매출 CAGR 필터 미적용**: 코드에 ≥10% 기준이 있으나 통과로 고정 → **현재 필터에 성장 축이 아예 없다**. 성장성 좋은 회사를 거르지도, 못 거르지도 않는 상태.

- **⚠️ WACC의 일률적 Re**: CAPM 베타가 없어 모든 기업이 동일한 Re를 쓴다 → **업종별 위험이 WACC에 반영 안 됨**. 또 법인세율 25% 고정이라 면세·이월결손 기업은 세후 ROIC/WACC가 왜곡될 수 있음.

- **참고 — FCF/ROIC/WACC 수집 의존성**: 이 셋은 CFO·유형/무형자산취득·이자비용 등 추가 데이터가 있어야 계산된다. 데이터 없는 연도는 계산이 생략되므로, 빈 값이면 "0"이 아니라 "원천 데이터 부재"일 수 있음(`docs/코드검사_개선사항.md` 맥락).
